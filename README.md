# Legal Agent Kit

**Legal Agent Kit** — открытый набор инструкций, шаблонов, чек-листов, примеров и готовых промптов для AI-агентов, которые помогают системно подготовить черновики правовых документов для цифрового проекта.

Репозиторий предназначен для предпринимателей, разработчиков, продуктовых команд, no-code/low-code специалистов и AI-агентов, которые хотят быстро и последовательно собрать базовый комплект документов для сайта, SaaS-сервиса, PWA, Telegram/MAX-бота, API-сервиса, маркетплейс-интеграции, сервиса с оплатой или другого цифрового продукта.

> Материалы репозитория не являются юридической консультацией и не заменяют проверку юристом. Они помогают собрать данные о проекте, выбрать нужные документы, подготовить структурированные черновики и внедрить их в интерфейс сайта или сервиса.

---

## Что решает Legal Agent Kit

Legal Agent Kit помогает:

- определить, какие правовые документы нужны конкретному проекту;
- собрать данные о сервисе, владельце, пользователях, платежах, подписках и интеграциях;
- описать обработку персональных данных, cookies, аналитики, API-токенов и банковских данных;
- подготовить черновики документов в Markdown;
- сформировать рекомендации по размещению документов на сайте, в PWA, личном кабинете или боте;
- подготовить тексты чекбоксов для регистрации, оплаты, заявок, подписки и согласий;
- описать хранение пользовательских согласий;
- подготовить список вопросов для владельца проекта или юриста;
- провести аудит уже существующих документов;
- обновить документы после изменения функциональности;
- использовать единый подход при работе с Codex, ChatGPT и другими AI-агентами.

## Основные файлы

### `AGENTS.md`

Инструкция для AI-агента: как использовать репозиторий, какой порядок работы соблюдать, что можно делать, чего нельзя делать и какой результат должен быть получен.

### `LEGAL_AGENT_GUIDE.md`

Главное методическое руководство. Описывает порядок работы, типы проектов, логику выбора документов, требования к результату и финальную проверку.

### `LEGAL_PROJECT_QUESTIONNAIRE.md`

Анкета проекта. Помогает собрать данные о владельце сервиса, пользователях, регистрации, платежах, подписках, персональных данных, cookies, аналитике, интеграциях, API, ботах, банковских данных и поддержке.

### `docs/`

Методические материалы:

- `document-matrix.md` — матрица выбора документов;
- `interface-checklist.md` — проверка внедрения документов в интерфейс;
- `consent-storage.md` — рекомендации по хранению фактов согласия;
- `legal-disclaimer.md` — юридический дисклеймер;
- `how-to-use-with-codex.md` — как использовать kit с Codex;
- `how-to-use-with-chatgpt.md` — как использовать kit с ChatGPT.

### `prompts/`

Готовые промпты для Codex и других AI-агентов:

- `codex-create-legal-docs.md` — создать комплект документов;
- `codex-integrate-legal-pages.md` — внедрить страницы, ссылки и чекбоксы;
- `codex-audit-existing-legal-docs.md` — провести аудит существующих документов;
- `codex-update-legal-docs.md` — обновить документы после изменения функциональности.

### `templates/ru/`

Шаблоны документов на русском языке с плейсхолдерами. Они не являются финальными документами и должны адаптироваться под конкретный проект.

### `checklists/ru/`

Чек-листы для проверки сайта или сервиса перед запуском: регистрация, оплата, подписка, персональные данные, cookies, аналитика, API-токены и интеграции.

### `examples/ru/`

Примеры применения для трех частых сценариев:

- `saas-subscription` — SaaS с подпиской;
- `telegram-max-bot` — Telegram/MAX-бот;
- `simple-landing` — простой лендинг.

## Быстрый старт для Codex

### Вариант 1. Использовать репозиторий по ссылке

Передайте AI-агенту ссылку на репозиторий и описание проекта:

```text
Используй Legal Agent Kit как методическую базу:
https://github.com/USERNAME/legal-agent-kit

Нужно подготовить комплект правовых документов для проекта.

Проект: [описание проекта]
Домен: [домен]
Владелец: [реквизиты]
Модель: [B2B/B2C]
Оплата: [есть/нет, платежный провайдер]
Подписка: [есть/нет]
Данные: [какие данные собираются]
Интеграции: [банки, маркетплейсы, мессенджеры, API]
Бот: [есть/нет, Telegram/MAX]

Сначала заполни анкету проекта, затем определи нужные документы, затем подготовь черновики документов, список вопросов, риски и рекомендации по внедрению.
```

### Вариант 2. Подключить как папку в проект

Скопируйте репозиторий или его содержимое в проект:

```text
/docs/legal-kit/
```

Затем поставьте задачу агенту:

```text
В проекте есть /docs/legal-kit/.
Используй эти инструкции для подготовки правовых документов.

Сначала заполни анкету проекта.
Если данных не хватает — оставь плейсхолдеры и список вопросов.
Готовые документы положи в /docs/legal/generated/.
Не меняй рабочий код проекта без отдельного подтверждения.
```

### Вариант 3. Подключить как Git submodule

```bash
git submodule add https://github.com/USERNAME/legal-agent-kit docs/legal-kit
```

Так можно использовать одну актуальную версию набора инструкций в нескольких проектах.

## Рекомендуемый порядок работы агента

1. Изучить описание проекта, README, маршруты, страницы регистрации, оплаты, тарифов и личного кабинета.
2. Определить тип проекта и модель работы: B2B, B2C или смешанная.
3. Заполнить `LEGAL_PROJECT_QUESTIONNAIRE.md` или создать анкету проекта.
4. Выбрать комплект документов по `LEGAL_AGENT_GUIDE.md` и `docs/document-matrix.md`.
5. Подготовить матрицу документов.
6. Подготовить документы на основе шаблонов из `templates/ru/`.
7. Не выдумывать неизвестные данные: реквизиты, ИНН, адреса, тарифы, условия возврата, платежных провайдеров, сроки хранения.
8. Оставить плейсхолдеры для неизвестных данных.
9. Подготовить список вопросов к владельцу проекта.
10. Подготовить список юридических рисков.
11. Подготовить рекомендации по ссылкам, чекбоксам и хранению согласий.
12. Отметить, что документы требуют юридической проверки.

## Как выбирать документы

| Условие проекта | Обычно нужны документы |
|---|---|
| Есть форма заявки | `privacy.md`, `personal_data_consent.md` |
| Есть cookies или аналитика | `cookie_policy.md`, раздел в `privacy.md` |
| Есть регистрация | `terms.md`, `privacy.md`, `personal_data_consent.md` |
| Есть личный кабинет | `terms.md`, `privacy.md` |
| Есть онлайн-оплата | `offer.md`, `payment_and_refund.md`, `privacy.md` |
| Есть подписка, trial или льготный период | `subscription_rules.md`, `offer.md`, `payment_and_refund.md` |
| Есть рекламные сообщения | `marketing_consent.md`, раздел в `privacy.md` |
| Есть API или webhook для клиентов | `api_terms.md`, `data_processing.md` |
| Пользователь передает API-токены | `data_processing.md`, раздел в `privacy.md` |
| Есть банковские данные | `bank_data_processing.md`, `privacy.md`, `personal_data_consent.md` |
| Есть Telegram/MAX-бот | `bot_terms.md`, `privacy.md`, `personal_data_consent.md` |
| Есть поддержка с условиями | `support_rules.md` |

## Что важно внедрить в интерфейс

Для большинства проектов нужно предусмотреть:

- ссылки на правовые документы в футере сайта;
- ссылки на документы в Telegram/MAX-боте, если бот используется;
- отдельные чекбоксы согласий на форме регистрации;
- согласие на обработку персональных данных на формах заявок;
- ссылку на оферту и правила оплаты перед оплатой;
- правила подписки перед подключением тарифа, trial или автопродления;
- отдельное необязательное согласие на рекламные сообщения;
- страницу с актуальными редакциями документов;
- фиксацию даты, версии документа, IP-адреса, user-agent и источника согласия;
- понятный контакт для обращений по персональным данным, оплате и возвратам.

Пример чекбоксов для регистрации:

```text
[ ] Я принимаю Пользовательское соглашение
[ ] Я ознакомлен с Политикой конфиденциальности
[ ] Я даю согласие на обработку персональных данных
```

Пример согласия на рекламные сообщения:

```text
[ ] Я согласен получать информационные и рекламные сообщения
```

Пример текста перед оплатой:

```text
Нажимая «Оплатить», я принимаю условия Публичной оферты и Правила оплаты и возврата.
```

Пример текста для первого сообщения бота:

```text
Перед использованием сервиса ознакомьтесь с Пользовательским соглашением и Политикой конфиденциальности. Продолжая работу с ботом, вы подтверждаете согласие с условиями сервиса.
```

## Ограничения

Legal Agent Kit не должен использоваться для автоматической публикации документов без проверки.

AI-агент не должен:

- выдавать документы как юридически финальные;
- обещать соответствие законодательству без проверки юристом;
- копировать документы с чужих сайтов;
- подставлять выдуманные реквизиты;
- выдумывать тарифы, платежных провайдеров, условия возврата или сроки хранения;
- смешивать согласие на обработку персональных данных с пользовательским соглашением;
- делать согласие на рекламные сообщения обязательным без необходимости;
- удалять важные ограничения ответственности;
- игнорировать особенности юрисдикции и модели бизнеса;
- игнорировать фактическую работу сервиса.

## Юридический дисклеймер

Материалы репозитория предоставляются в информационных и методических целях.

Они не являются юридической консультацией, правовым заключением или гарантией соответствия законодательству.

Перед публикацией документов необходимо провести юридическую проверку с учетом конкретной юрисдикции, модели бизнеса, состава персональных данных, используемых платежных сервисов, внешних интеграций и фактической работы проекта.
