

# Legal Agent Kit

**Legal Agent Kit** — открытый набор инструкций, шаблонов, чек-листов и промптов для AI-агентов, которые помогают подготовить черновики правовых документов для сайта, SaaS-сервиса, PWA, бота, API-сервиса, маркетплейс-интеграции или другого цифрового продукта.

Репозиторий предназначен для предпринимателей, разработчиков, продуктовых команд, no-code/low-code специалистов и AI-агентов, которые хотят быстро и системно подготовить базовый комплект документов для нового проекта.

> Материалы репозитория не являются юридической консультацией и не заменяют проверку юристом. Они помогают собрать данные о проекте, выбрать нужные документы, подготовить структурированные черновики и внедрить их в интерфейс сайта или сервиса.

---

## Какие задачи решает репозиторий

Legal Agent Kit помогает:

- определить, какие правовые документы нужны конкретному проекту;
- собрать данные о сервисе, владельце, платежах, пользователях и обработке данных;
- подготовить черновики документов в Markdown;
- сформировать рекомендации по размещению документов на сайте;
- подготовить тексты чекбоксов для регистрации, оплаты и согласий;
- описать хранение пользовательских согласий;
- подготовить список вопросов для владельца проекта или юриста;
- использовать единый подход при работе с разными AI-агентами и Codex.

- корпоративный сайт;
- SaaS-сервис;
- PWA-приложение;
- Telegram-бот;
- MAX-бот;
- интернет-магазин;
- API-сервис;

- сервис уведомлений;

---

## Какие документы можно подготовить

В базовый комплект могут входить:

- Пользовательское соглашение;
- Политика конфиденциальности;
- Согласие на обработку персональных данных;
- Публичная оферта;
- Правила оплаты и возврата;
- Политика cookies;
- Правила подписки;
- Правила поддержки;
- Согласие на информационные и рекламные сообщения;
- Документ или раздел об обработке API-токенов, ключей доступа и данных интеграций;
- Документ или раздел об обработке банковских данных и данных, полученных через авторизацию в банке.

Набор документов зависит от функций конкретного проекта: регистрации, оплаты, подписки, аналитики, внешних интеграций, ботов, API и состава обрабатываемых данных.

## Основные файлы

### `AGENTS.md`

Краткая инструкция для AI-агента: как использовать репозиторий, что можно делать, чего нельзя делать и какой результат должен быть получен.

### `LEGAL_AGENT_GUIDE.md`

Главное методическое руководство для агента. Описывает порядок работы, типы проектов, матрицу документов, требования к результату и финальную проверку.

### `LEGAL_PROJECT_QUESTIONNAIRE.md`

Анкета проекта. Помогает собрать данные о владельце сервиса, пользователях, платежах, подписках, персональных данных, интеграциях и поддержке.

### `templates/ru/`

Шаблоны документов на русском языке с плейсхолдерами. Они не являются финальными документами и должны адаптироваться под конкретный проект.

### `prompts/`

Готовые промпты для Codex и других AI-агентов:

- создание документов;
- внедрение документов в проект;
- аудит существующих документов;
- обновление документов при изменении функциональности.

### `checklists/ru/`

Чек-листы для проверки сайта или сервиса перед запуском: регистрация, оплата, подписки, персональные данные, cookies, API-токены.

### `examples/ru/`

Примеры применения для разных типов проектов.

## Быстрый старт для Codex

### Вариант 1. Использовать репозиторий по ссылке

Передайте AI-агенту ссылку на этот репозиторий и описание проекта:

```text
Используй Legal Agent Kit как методическую базу:
https://github.com/USERNAME/legal-agent-kit

Нужно подготовить комплект правовых документов для проекта.

Проект: [описание проекта]
Домен: [домен]
Владелец: [реквизиты]
Модель: [B2B/B2C]
Оплата: [есть/нет, платежный провайдер]
Подписка: [есть/нет]
Данные: [какие данные собираются]
Интеграции: [банки, маркетплейсы, мессенджеры, API]

Сначала заполни анкету проекта, затем определи нужные документы, затем подготовь черновики документов и рекомендации по внедрению.
```

### Вариант 2. Подключить как папку в проект

Скопируйте репозиторий или его содержимое в проект:

```text
/docs/legal-kit/
```

Затем поставьте задачу агенту:

```text
В проекте есть /docs/legal-kit/.
Используй эти инструкции для подготовки правовых документов.

Сначала заполни анкету проекта.
Если данных не хватает — оставь плейсхолдеры и список вопросов.
Готовые документы положи в /docs/legal/generated/.
```

### Вариант 3. Подключить как Git submodule

```bash
git submodule add https://github.com/USERNAME/legal-agent-kit docs/legal-kit
```

Так можно использовать одну актуальную версию набора инструкций в нескольких проектах.

## Рекомендуемый порядок работы агента

1. Изучить описание проекта, README, маршруты, страницы регистрации, оплаты и личного кабинета.
2. Заполнить `LEGAL_PROJECT_QUESTIONNAIRE.md`.
3. Определить тип проекта и модель работы: B2B, B2C или смешанная.
4. Выбрать комплект документов по матрице.
5. Подготовить документы на основе шаблонов.
6. Не выдумывать неизвестные данные: реквизиты, ИНН, адреса, тарифы, условия возврата, платежных провайдеров.
7. Оставить плейсхолдеры для неизвестных данных.
8. Подготовить список вопросов к владельцу проекта.
9. Подготовить рекомендации по ссылкам, чекбоксам и хранению согласий.
10. Отметить, что документы требуют юридической проверки.

## Что важно внедрить в интерфейс

Для большинства проектов с регистрацией и оплатой нужно предусмотреть:

- ссылки на правовые документы в футере сайта;
- отдельные чекбоксы согласий на форме регистрации;
- ссылку на оферту и правила оплаты перед оплатой;
- страницу с актуальными редакциями документов;
- фиксацию даты, версии документа, IP-адреса и user-agent при принятии согласий;
- понятный контакт для обращений по персональным данным и возвратам.

Пример чекбоксов для регистрации:

```text
[ ] Я принимаю Пользовательское соглашение
[ ] Я ознакомлен с Политикой конфиденциальности
[ ] Я даю согласие на обработку персональных данных
```

Пример текста перед оплатой:

```text
Нажимая «Оплатить», я принимаю условия Публичной оферты и Правила оплаты и возврата.
```

## Ограничения

Legal Agent Kit не должен использоваться для автоматической публикации документов без проверки.

AI-агент не должен:

- выдавать документы как юридически финальные;
- копировать документы с чужих сайтов;
- подставлять выдуманные реквизиты;
- смешивать согласие на обработку персональных данных с пользовательским соглашением;
- делать согласие на рекламные сообщения обязательным без необходимости;
- удалять важные ограничения ответственности;
- игнорировать особенности юрисдикции и модели бизнеса.

## Юридический дисклеймер

Материалы репозитория предоставляются в информационных и методических целях.

Они не являются юридической консультацией, правовым заключением или гарантией соответствия законодательству.

Перед публикацией документов необходимо провести юридическую проверку с учетом конкретной юрисдикции, модели бизнеса, состава персональных данных, используемых платежных сервисов, внешних интеграций и фактической работы проекта.
