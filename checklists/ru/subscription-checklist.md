

# Чек-лист подписки

## Назначение

Этот чек-лист помогает проверить, что подписочная модель сайта, SaaS-сервиса, PWA, личного кабинета или бота юридически и продуктово описана достаточно понятно для пользователя.

Чек-лист используется перед запуском подписки, изменением тарифов, добавлением пробного периода, льготного периода, автопродления, отключения подписки или правил возврата.

---

## 1. Когда использовать чек-лист

Используйте этот чек-лист, если в проекте есть:

- платный доступ на период;
- тарифы на месяц, год или фиксированное количество дней;
- пробный период;
- бесплатный тариф с ограничениями;
- льготный период после окончания оплаты;
- ручное продление подписки;
- автоматическое продление подписки;
- ограничение функций по тарифам;
- расчет тарифа по объему использования;
- отключение доступа после окончания подписки;
- возвраты по подписке.

## 2. Документы, которые обычно нужны для подписки

| Документ | Нужен | Комментарий |
|---|---|---|
| Публичная оферта | [да / нет / уточнить] | Нужна при платной подписке |
| Правила подписки | [да / нет / уточнить] | Нужны для периода доступа, продления, отключения и ограничений |
| Правила оплаты и возврата | [да / нет / уточнить] | Нужны при оплате подписки |
| Пользовательское соглашение | [да / нет / уточнить] | Нужно для правил использования сервиса |
| Политика конфиденциальности | [да / нет / уточнить] | Нужна при обработке данных пользователя |
| Согласие на обработку персональных данных | [да / нет / уточнить] | Если собираются персональные данные |
| Согласие на рекламные сообщения | [да / нет / уточнить] | Если есть маркетинговые рассылки |

## 3. Описание тарифов

| Проверка | Статус | Комментарий |
|---|---|---|
| У каждого тарифа есть название | [ ] | Названия должны совпадать на сайте, в оферте и правилах подписки |
| У каждого тарифа есть стоимость | [ ] | Цена, валюта, период |
| Указан период действия тарифа | [ ] | Например: 30 дней, месяц, год |
| Описаны возможности тарифа | [ ] | Какие функции доступны |
| Описаны ограничения тарифа | [ ] | Лимиты, пользователи, объемы, интеграции |
| Описано, что происходит при превышении лимитов | [ ] | Блокировка, доплата, ограничение функций |
| Описано, можно ли менять тариф | [ ] | Повышение, понижение, перерасчет |
| Описано, когда новый тариф вступает в силу | [ ] | Сразу, после окончания периода, после оплаты |
| Тарифы на сайте не противоречат оферте | [ ] | Проверить вручную |
| Тарифы в интерфейсе не противоречат правилам подписки | [ ] | Проверить вручную |

Пример описания тарифа:

```text
Тариф «Базовый» предоставляет доступ к основным функциям сервиса на 30 календарных дней с даты оплаты. Ограничения тарифа указаны на странице тарифов и в Правилах подписки.
```

## 4. Бесплатный тариф

Если в проекте есть бесплатный тариф, проверьте:

| Проверка | Статус | Комментарий |
|---|---|---|
| Бесплатный тариф описан отдельно | [ ] | Не путать с пробным периодом |
| Указаны ограничения бесплатного тарифа | [ ] | Функции, лимиты, пользователи, период |
| Указано, может ли сервис изменить бесплатный тариф | [ ] | Например, изменить лимиты или закрыть тариф |
| Указано, когда доступ может быть ограничен | [ ] | Нарушения, неактивность, прекращение тарифа |
| Указано, можно ли перейти на платный тариф | [ ] | И как это сделать |
| Бесплатный тариф не обещает платные функции | [ ] | Нет противоречий в маркетинговых блоках |

---

## 5. Пробный период

Если есть пробный период, он должен быть описан отдельно от бесплатного тарифа.

| Проверка | Статус | Комментарий |
|---|---|---|
| Указана длительность пробного периода | [ ] | Например: 7, 14, 30 дней |
| Указано, когда начинается пробный период | [ ] | После регистрации, после подключения, после подтверждения email |
| Указано, какие функции доступны в пробном периоде | [ ] | Полный доступ или ограничения |
| Указано, что будет после окончания пробного периода | [ ] | Ограничение доступа, предложение оплатить, автосписание |
| Указано, можно ли получить пробный период повторно | [ ] | Например: один раз на пользователя/организацию |
| Указано, нужна ли карта для пробного периода | [ ] | Если используется автосписание после trial |
| Указано, как отказаться от продолжения | [ ] | Если после trial есть автопродление |
| Пробный период не противоречит странице тарифов | [ ] | Проверить сайт и документы |
| Принятие условий пробного периода фиксируется | [ ] | Если trial является частью подписки |

Пример текста:

```text
Пробный период предоставляется один раз для одного пользователя или организации и действует в течение [УТОЧНИТЬ] календарных дней. После окончания пробного периода доступ к платным функциям ограничивается, если пользователь не оплатил подписку.
```

## 7. Продление подписки

| Проверка | Статус | Комментарий |
|---|---|---|
| Описан способ продления | [ ] | Ручное или автоматическое |

## 8. Автопродление

Если есть автоматические повторные списания, условия должны быть особенно понятными.

| Проверка | Статус | Комментарий |
|---|---|---|
| Автопродление явно указано до оплаты | [ ] | Пользователь должен видеть это до списания |
| Указана периодичность списаний | [ ] | Например: каждые 30 дней, ежемесячно |
| Указана сумма или порядок расчета суммы | [ ] | Фиксированная или расчетная |
| Указано, с какого платежного средства списываются деньги | [ ] | Если применимо |
| Указано, как отключить автопродление | [ ] | В личном кабинете или через поддержку |
| Указано, за какой срок нужно отключить автопродление | [ ] | Если применимо |
| Указано, что будет при неуспешном списании | [ ] | Повтор, льготный период, блокировка |
| Фиксируется согласие на автопродление | [ ] | Версия, дата, IP, user-agent |
| Пользователь может увидеть статус автопродления | [ ] | В личном кабинете |
| Нужно ли повторное согласие старых пользователей | [ ] | Если автопродление добавлено позже |

Пример текста:

```text
Если для тарифа включено автопродление, подписка автоматически продлевается на следующий период до ее отключения пользователем. Условия списаний, порядок отключения и последствия неуспешного платежа описаны в Правилах подписки.
```

## 9. Льготный период

Если после окончания оплаченного периода сервис дает временный доступ, это нужно описать.

| Проверка | Статус | Комментарий |
|---|---|---|
| Указано, есть ли льготный период | [ ] | Да/нет |
| Указана длительность льготного периода | [ ] | Например: 3, 5, 7 дней |
| Указано, какие функции доступны в льготный период | [ ] | Полный или ограниченный доступ |
| Указано, что происходит после окончания льготного периода | [ ] | Приостановка, блокировка, архив |
| Указано, можно ли оплатить в льготный период | [ ] | И как считается новый период |
| Льготный период отображается в личном кабинете | [ ] | Если есть ЛК |
| Пользователь получает уведомление о льготном периоде | [ ] | Если предусмотрено |

Пример текста:

```text
После окончания оплаченного периода сервис может предоставить льготный период продолжительностью [УТОЧНИТЬ] календарных дней. В течение льготного периода доступ может быть ограничен. Если подписка не будет оплачена до окончания льготного периода, доступ к платным функциям приостанавливается.
```

## 15. Хранение принятия правил подписки

При подключении или оплате подписки рекомендуется фиксировать:

```text
offer
payment_and_refund
subscription_rules
```

Если есть автопродление, обязательно зафиксировать текст подтверждения автопродления.

Рекомендуемые поля:

| Поле | Нужно | Комментарий |
|---|---|---|
| `user_id` | если есть | ID пользователя |
| `account_id` | если есть | ID аккаунта/tenant |
| `subscription_id` | желательно | ID подписки |
| `payment_id` | если есть | ID платежа |
| `tariff_id` | если есть | ID тарифа |
| `document_type` | да | `offer`, `payment_and_refund`, `subscription_rules` |
| `document_version` | да | Версия документа |
| `document_date` | желательно | Дата редакции |
| `document_url` | желательно | URL документа |
| `checkbox_text` | желательно | Текст подтверждения |
| `accepted_at` | да | Дата и время принятия |
| `ip_address` | желательно | IP пользователя |
| `user_agent` | желательно | Браузер/клиент |
| `source` | да | `payment`, `profile`, `bot` |
| `meta_json` | если нужно | Тариф, период, сумма, автопродление |

Пример `meta_json`:

```json
{
  "tariff": "pro",
  "amount": 2990,
  "currency": "RUB",
  "period_days": 30,
  "auto_renewal": false,
  "trial_days": 14,
  "grace_period_days": 5
}
```

## 16. Повторное согласие

Повторное согласие может понадобиться, если:

- добавлено автопродление;
- изменились тарифы;
- изменились правила возврата;
- изменился период подписки;
- изменились ограничения тарифа;
- изменился момент оказания услуги;
- появились новые платные функции;
- изменились существенные условия оферты;
- изменились условия обработки персональных данных.

| Проверка | Статус | Комментарий |
|---|---|---|
| Определено, нужно ли повторное согласие | [ ] | Проверить с юристом |
| Указано, какие документы изменились | [ ] | Оферта, правила подписки, правила оплаты |
| Новая версия документов опубликована | [ ] | Дата и версия |
| Пользователь уведомлен об изменениях | [ ] | Если требуется |
| Согласие на новую редакцию фиксируется | [ ] | В журнале согласий |

## 17. Типовые ошибки

| Ошибка | Почему это плохо |
|---|---|
| Не указана дата окончания подписки | Пользователь не понимает срок доступа |
| Пробный период не описан в документах | Возникают споры после trial |
| Автопродление спрятано в мелком тексте | Риск претензий по повторным списаниям |
| Нет правил отключения подписки | Пользователь не понимает, как остановить списания |
| Нет правил возврата | Непонятно, что делать при споре |
| Тарифы в интерфейсе отличаются от оферты | Противоречие документов и сайта |
| Нет фиксации принятия правил подписки | Сложно доказать согласие пользователя |
| Не описан льготный период | Пользователь не понимает статус доступа |
| Не описано, что происходит с данными после окончания подписки | Риск претензий при блокировке или удалении данных |
| Рекламные уведомления смешаны с сервисными | Риск по согласию на рекламу |
