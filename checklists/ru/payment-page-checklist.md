

# Чек-лист страницы оплаты

## Назначение

Этот чек-лист помогает проверить, что страница оплаты сайта, SaaS-сервиса, PWA, личного кабинета или бота содержит необходимые правовые ссылки, корректно описывает условия оплаты, подписки, возврата и фиксирует принятие пользователем документов.

Чек-лист используется перед запуском оплаты, после подключения нового платежного провайдера, изменения тарифов, добавления подписки, пробного периода или автопродления.

---

## 1. Когда использовать чек-лист

Используйте этот чек-лист, если в проекте есть:

- онлайн-оплата;
- платные тарифы;
- подписка;
- пробный период;
- автопродление;
- ручное продление доступа;
- разовая покупка услуги;
- оплата через банк или платежного провайдера;
- фискализация и выдача чеков;
- оплата в Telegram/MAX-боте;
- оплата в PWA или личном кабинете.

## 5. Текст принятия условий перед оплатой

Перед кнопкой оплаты или рядом с ней рекомендуется разместить явный текст.

### Разовая оплата

```text
Нажимая «Оплатить», я принимаю условия Публичной оферты и Правила оплаты и возврата.
```

### Оплата подписки

```text
Нажимая «Оплатить», я принимаю условия Публичной оферты, Правила подписки и Правила оплаты и возврата.
```

### Оплата с обработкой персональных данных

```text
Нажимая «Оплатить», я принимаю условия Публичной оферты, Правила оплаты и возврата, а также соглашаюсь с Политикой конфиденциальности.
```

### Подписка с автопродлением

```text
Нажимая «Оплатить», я принимаю условия Публичной оферты, Правила подписки и Правила оплаты и возврата. Подписка продлевается автоматически до ее отключения пользователем.
```

## 14. Хранение принятия документов при оплате

При оплате рекомендуется фиксировать принятие:

```text
offer
payment_and_refund
subscription_rules
```

Если есть персональные данные:

```text
privacy
personal_data_consent
```

Рекомендуемые поля:

| Поле | Нужно | Комментарий |
|---|---|---|
| `user_id` | если есть | ID пользователя |
| `account_id` | если есть | ID аккаунта/tenant |
| `payment_id` | желательно | ID платежа в системе |
| `order_id` | если есть | ID заказа или подписки |
| `document_type` | да | `offer`, `payment_and_refund`, `subscription_rules` |
| `document_version` | да | Версия документа |
| `document_date` | желательно | Дата редакции |
| `document_url` | желательно | URL документа |
| `checkbox_text` | желательно | Текст перед оплатой или чекбокс |
| `accepted_at` | да | Дата и время принятия |
| `ip_address` | желательно | IP пользователя |
| `user_agent` | желательно | Браузер/клиент |
| `source` | да | `payment` |
| `meta_json` | если нужно | Тариф, сумма, валюта, период |

Пример `meta_json`:

```json
{
  "tariff": "business",
  "amount": 1990,
  "currency": "RUB",
  "period_days": 30,
  "payment_provider": "tbank"
}
```

## 16. Типовые ошибки

| Ошибка | Почему это плохо |
|---|---|
| Нет ссылки на оферту перед оплатой | Пользователь не принимает условия договора до оплаты |
| Нет правил возврата | Непонятно, как обрабатываются возвраты |
| Тарифы на сайте отличаются от оферты | Возникает противоречие |
| Не указано, когда услуга считается оказанной | Сложности с возвратами |
| Автопродление скрыто или описано неявно | Риск претензий по повторным списаниям |
| Не фиксируется принятие оферты | Сложно доказать акцепт |
| Реквизиты в оферте устарели | Документ может быть неприменимым |
| Платежный провайдер не описан в политике | Не описана передача данных третьей стороне |
| Ошибки платежей показывают внутренние данные | Риск безопасности |
| В логах сохраняются токены или секреты | Критичный риск безопасности |
