## 2. Термины

2.1. **API** — программный интерфейс сервиса, позволяющий внешним системам взаимодействовать с функциями сервиса через HTTP-запросы, webhook, токены доступа или иные технические механизмы.

2.2. **API-клиент** — внешняя система, приложение, сайт, сервер, скрипт, интеграция или иной программный компонент, который обращается к API сервиса.

2.3. **Пользователь API** — пользователь, организация, разработчик или интегратор, который получил доступ к API и использует его самостоятельно или от имени клиента.

2.4. **API-ключ / токен доступа** — ключ, Bearer token, OAuth token, service token или иной идентификатор, используемый для авторизации в API.

2.5. **Webhook** — механизм доставки событий из сервиса на URL, указанный пользователем, или механизм приема событий сервисом от внешних систем.

2.6. **Документация API** — описание методов, параметров, форматов запросов и ответов, кодов ошибок, лимитов и иных технических требований.

2.7. **Лимит API** — ограничение количества запросов, частоты обращений, объема данных, числа webhook-событий или иных параметров использования API.
