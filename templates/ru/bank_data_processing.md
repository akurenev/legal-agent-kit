## 2. Термины

2.1. **Сервис** — сайт, приложение, PWA, личный кабинет, бот, API и иные компоненты **[НАЗВАНИЕ СЕРВИСА]**.

2.2. **Пользователь** — лицо, которое использует сервис и подключает банковскую интеграцию.

2.3. **Банк** — кредитная организация, платежная организация, финансовая организация, API-провайдер или иной внешний сервис, предоставляющий доступ к банковским или финансовым данным.

2.4. **Банковская интеграция** — подключение сервиса к банку или финансовому сервису для получения, обработки или передачи банковских данных.

2.5. **Банковские данные** — данные о счетах, операциях, остатках, выписках, контрагентах, платежах, назначениях платежей, реквизитах, статусах операций и иные финансовые данные, полученные через банковскую интеграцию.

2.6. **OAuth-доступ** — механизм авторизации, при котором пользователь разрешает сервису получать доступ к определенным банковским данным или действиям в банке.

2.7. **Токен банка** — access token, refresh token, API key, client secret, webhook secret или иной идентификатор, используемый для банковской интеграции.

## 7. Получение доступа к банковским данным

7.1. Пользователь может подключить банк через личный кабинет, приложение, PWA, бота, OAuth-сценарий, API или иной интерфейс сервиса.

## 12. Отключение банка

12.1. Пользователь может отключить банк, если такая функция предусмотрена интерфейсом сервиса.

12.2. После отключения банка сервис прекращает новые обращения к банковскому API, если иное не требуется для завершения уже начатой операции, выполнения законных обязанностей, безопасности или защиты прав.

12.3. Отключение банка может привести к недоступности функций, связанных с банковской интеграцией.

12.4. Пользователь также может отозвать доступ на стороне банка, если банк предоставляет такую возможность.

12.5. Если банк отзывает доступ, блокирует подключение, изменяет API или ограничивает права, интеграция может перестать работать.

- повторить запрос;
- отложить обработку;

- уведомить пользователя;
- запросить повторное подключение банка;
- отключить интеграцию;
- обратиться к пользователю за уточнением настроек.

16.4. Пользователь понимает, что работа банковской интеграции зависит от банка, прав доступа, лимитов, состояния банковского аккаунта, настроек безопасности и качества переданных разрешений.

## 17. Логи банковской интеграции

17.1. Для диагностики и поддержки сервис может сохранять логи банковской интеграции.

17.2. Логи могут содержать:

- дату и время запроса;
- тип банковской интеграции;
- идентификатор подключения;
- endpoint банка или API;
- HTTP-код ответа;
- статус операции;
- текст ошибки;
- request_id / correlation_id;
- длительность запроса;
- технические метаданные.
