

# Пример чек-листа внедрения документов: Telegram/MAX-бот

## Назначение примера

Этот пример показывает, как AI-агент может подготовить чек-лист внедрения правовых документов в Telegram/MAX-бота, WebApp/PWA, сайт, оплату, поддержку и настройки интеграций.

Пример основан на демонстрационных файлах:

```text
examples/ru/telegram-max-bot/project-questionnaire.example.md
examples/ru/telegram-max-bot/legal-documents-matrix.example.md
```

Все данные ниже демонстрационные. Перед публикацией документов и изменениями интерфейса нужно провести юридическую и техническую проверку.

---

## 1. Что внедряем

| Элемент | Статус | Комментарий |
|---|---|---|
| Публичные страницы документов | нужно | Документы должны открываться без авторизации |
| Ссылки на документы в первом сообщении бота | нужно | Пользователь должен увидеть документы до использования функций |
| Кнопка принятия условий | желательно | Особенно если бот регистрирует пользователя или принимает персональные данные |
| Пункт меню «Правовые документы» | желательно | Чтобы пользователь мог вернуться к документам позже |
| Согласие на обработку персональных данных | нужно | При регистрации, заявках, телефоне, email, поддержке |
| Отдельное рекламное согласие | уточнить | Нужно, если есть рекламные сообщения |
| Ссылки на оферту перед оплатой | нужно, если есть оплата | До перехода к платежу |
| Правила подписки перед оплатой | нужно, если есть подписка | До подключения тарифа/trial |
| Предупреждение при отправке файлов | желательно | Не отправлять пароли, токены, банковские карты |
| Хранение согласий | нужно | Версия, дата, источник, messenger ID или user_id |

## 2. Публичные URL документов

Рекомендуемые URL для сайта или WebApp:

| Документ | URL | Нужен |
|---|---|---|
| Пользовательское соглашение | `/legal/terms` | да |
| Политика конфиденциальности | `/legal/privacy` | да |
| Согласие на обработку персональных данных | `/legal/personal-data-consent` | да |
| Правила использования бота | `/legal/bot-terms` | да |
| Правила поддержки | `/legal/support-rules` | да |
| Публичная оферта | `/legal/offer` | если есть оплата |
| Правила оплаты и возврата | `/legal/payment-and-refund` | если есть оплата |
| Правила подписки | `/legal/subscription-rules` | если есть подписка/trial |
| Политика cookies | `/legal/cookie-policy` | если есть сайт/WebApp/PWA/cookies |
| Согласие на рекламные сообщения | `/legal/marketing-consent` | если есть рекламные сообщения |
| Правила обработки API-токенов | `/legal/data-processing` | если есть CRM/API/webhook |
| Правила обработки банковских данных | `/legal/bank-data-processing` | если есть банк |
| Условия использования API | `/legal/api-terms` | если есть клиентский API |

Если у проекта статический сайт, можно использовать HTML-файлы в `/public/legal/`.

## 3. Первое сообщение бота

### Что проверить

| Проверка | Статус | Комментарий |
|---|---|---|
| При `/start` бот показывает краткое описание сервиса | [ ] | Пользователь понимает, что делает бот |
| При `/start` есть ссылки на документы | [ ] | terms, privacy, personal data consent, bot terms |
| Ссылки открываются без авторизации | [ ] | Документы должны быть доступны до регистрации |
| Есть понятная кнопка принятия условий | [ ] | Желательно для регистрации и персональных данных |
| Без принятия условий закрытые функции недоступны | [ ] | Если функции требуют согласия |
| Факт принятия условий сохраняется | [ ] | Версия, дата, источник `bot` |

### Пример сообщения

```text
Здравствуйте! Это Demo Bot Service.

Бот помогает отправлять заявки, получать уведомления и обращаться в поддержку.

Перед использованием ознакомьтесь с документами:
— Пользовательское соглашение
— Политика конфиденциальности
— Согласие на обработку персональных данных
— Правила использования бота

Нажимая «Принять условия», вы подтверждаете согласие с документами и даете согласие на обработку персональных данных.
```

### Рекомендуемые кнопки

```text
[Принять условия]
[Правовые документы]
[Поддержка]
```

## 5. Регистрация через бота

### Что проверить

| Проверка | Статус | Комментарий |
|---|---|---|
| Бот объясняет, какие данные запрашивает | [ ] | Например: телефон, email, имя |
| Бот объясняет, зачем нужны данные | [ ] | Регистрация, связь, поддержка, чеки |
| Есть ссылка на Политику конфиденциальности | [ ] | До передачи данных |
| Есть согласие на обработку персональных данных | [ ] | До регистрации |
| Фиксируется факт согласия | [ ] | user_consents или аналог |
| Можно отменить регистрацию | [ ] | Команда или кнопка |
| Не запрашиваются лишние данные | [ ] | Только необходимые поля |

### Пример текста перед запросом телефона

```text
Для регистрации нужен номер телефона. Он используется для связи с вами, обработки обращений и защиты аккаунта.

Продолжая регистрацию, вы соглашаетесь с Политикой конфиденциальности и даете согласие на обработку персональных данных.
```

## 6. WebApp/PWA внутри бота

Если бот открывает WebApp или PWA, применяются требования как к обычному веб-интерфейсу.

| Проверка | Статус | Комментарий |
|---|---|---|
| В WebApp есть ссылки на документы | [ ] | В футере, меню или настройках |
| Регистрация в WebApp содержит чекбоксы | [ ] | terms, privacy, personal_data_consent |
| Чекбоксы не отмечены заранее | [ ] | Пользователь должен сам подтвердить |
| Есть cookie policy, если используются cookies/localStorage/analytics | [ ] | Особенно для PWA |
| Есть текст согласия перед оплатой | [ ] | Если WebApp принимает оплату |
| Согласия из WebApp фиксируются отдельно | [ ] | source = webapp / registration / payment |
| WebApp не хранит секреты в localStorage | [ ] | Токены, payment secret, bot token нельзя хранить открыто |

## 7. Форма заявки или заказа в боте

| Проверка | Статус | Комментарий |
|---|---|---|
| Бот сообщает, какие данные будут отправлены | [ ] | Имя, телефон, текст заявки, файлы |
| Есть ссылка на Политику конфиденциальности | [ ] | До отправки заявки |
| Есть текст согласия на обработку данных | [ ] | До отправки заявки |
| Пользователь может отменить ввод | [ ] | Кнопка «Отмена» |
| Вложения принимаются только если они нужны | [ ] | Не собирать лишние файлы |
| Есть предупреждение не отправлять секреты | [ ] | Пароли, токены, карты, приватные ключи |
| Заявка сохраняется с user_id/messenger_id | [ ] | Для истории и поддержки |
| Срок хранения заявок определен | [ ] | В privacy/bot_terms |

### Пример предупреждения перед файлом

```text
Не отправляйте через бот пароли, полные API-токены, банковские карты, платежные секреты и приватные ключи. Если для диагностики нужен доступ, поддержка предложит безопасный способ передачи.
```

## 10. Рекламные сообщения через бота

Если бот отправляет рекламные или маркетинговые сообщения, нужен отдельный сценарий согласия.

| Проверка | Статус | Комментарий |
|---|---|---|
| Есть отдельный документ marketing_consent | [ ] | Если есть реклама |
| Рекламное согласие не обязательно для регистрации | [ ] | Если реклама не часть услуги |
| Есть отдельная кнопка согласия | [ ] | Например: «Получать новости и предложения» |
| Есть команда или кнопка отписки | [ ] | Например: `/unsubscribe` |
| Отказ от рекламы фиксируется | [ ] | event_type = revoked |
| После отписки сервисные уведомления продолжают приходить | [ ] | Если они нужны для работы сервиса |

### Пример текста согласия

```text
Хотите получать новости, полезные материалы и специальные предложения? Это необязательно. Вы сможете отписаться в любой момент.
```

### Пример кнопок

```text
[Получать новости и предложения]
[Не получать]
```

## 19. Итоговый вывод агента

Для Demo Bot Service нужно внедрить правовые документы в три уровня:

1. Публичные страницы документов на сайте или WebApp.
2. Ссылки и принятие условий внутри Telegram/MAX-бота.
3. Хранение факта согласий с версиями документов.

Минимально нужно добавить:

- ссылки на документы в `/start`;
- кнопку «Принять условия»;
- пункт меню «Правовые документы»;
