

# Пример матрицы правовых документов: Telegram/MAX-бот

## Назначение примера

Этот пример показывает, как AI-агент может определить комплект правовых документов для сервиса, который работает через Telegram-бота, MAX-бота или другой мессенджер.

Пример основан на демонстрационной анкете проекта:

```text
examples/ru/telegram-max-bot/project-questionnaire.example.md
```

Все данные ниже демонстрационные. Итоговый комплект документов для реального проекта должен быть проверен владельцем проекта и юристом.

---

## 2. Итоговая матрица документов

| Документ | Нужен | Причина | Какие данные нужны |
|---|---|---|---|
| Пользовательское соглашение | да | Пользователь использует сервис через бота, передает данные, получает доступ к функциям и уведомлениям | Название сервиса, владелец, реквизиты, правила использования, блокировка, ограничения |
| Политика конфиденциальности | да | Бот обрабатывает Telegram/MAX ID, chat_id, сообщения, контакты, технические данные и обращения | Реквизиты оператора, состав данных, цели обработки, сроки хранения, третьи лица, мессенджеры |
| Согласие на обработку персональных данных | да | Есть регистрация, сообщения, поддержка, возможно телефон/email и WebApp | Перечень данных, цели обработки, срок действия согласия, порядок отзыва |
| Правила использования бота | да | Нужен специальный документ для команд, сообщений, уведомлений, мессенджера и ограничений | Название бота, ссылки, функции, команды, данные мессенджера, отключение бота |
| Правила поддержки | да | Бот используется для обращений, поддержки и взаимодействия с оператором | Каналы поддержки, время работы, сроки реакции, что входит и не входит в поддержку |
| Публичная оферта | уточнить | Нужна, если через бота продаются услуги, тарифы, подписки или платные функции | Тарифы, стоимость, порядок акцепта, момент оказания услуги, возвраты |
| Правила оплаты и возврата | уточнить | Нужны, если есть оплата через бота, сайт, WebApp или платежную ссылку | Платежный провайдер, чек, статусы платежей, условия возврата |
| Правила подписки | уточнить | Нужны, если бот работает по подписке, тарифам, trial или льготному периоду | Период подписки, trial, автопродление, окончание доступа, смена тарифа |
| Политика cookies | уточнить | Нужна, если есть сайт, лендинг, WebApp, PWA, cookies или аналитика | Список cookies, аналитика, пиксели, cookie-баннер, отключение cookies |
| Согласие на информационные и рекламные сообщения | уточнить | Нужно, если бот/email/push отправляет рекламные или маркетинговые сообщения | Каналы рассылки, виды сообщений, способ отписки, фиксация согласия |
| Правила обработки API-токенов и данных интеграций | уточнить | Нужны, если пользователь подключает CRM, API, webhook или внешние сервисы | Типы токенов, хранение, удаление, цели, внешние сервисы |
| Правила обработки банковских данных | нет / уточнить | Не нужны, если бот не получает банковские данные | Нужны при банковских уведомлениях, выписках, остатках, операциях |
| Условия использования API | уточнить | Нужны, если сервис предоставляет API клиентам или внешним системам | Авторизация, лимиты, методы, webhook, логи, блокировка доступа |

## 3. Базовый комплект документов для бота без оплаты

Если бот бесплатный, без подписки, без рекламных сообщений и без пользовательских API-интеграций, минимальный комплект:

```text
terms.md
privacy.md
personal_data_consent.md
bot_terms.md
support_rules.md
```

Если есть публичный сайт или WebApp с cookies/аналитикой, добавить:

```text
cookie_policy.md
```

---

## 4. Комплект документов для бота с оплатой

Если бот принимает оплату, продает услуги, подключает тарифы или платные функции:

```text
terms.md
privacy.md
personal_data_consent.md
bot_terms.md
support_rules.md
offer.md
payment_and_refund.md
```

Если доступ предоставляется на период или есть подписка:

```text
subscription_rules.md
```

Если есть пробный период:

```text
subscription_rules.md
offer.md
```

---

## 5. Комплект документов для бота с рекламными сообщениями

Если бот отправляет рекламные, маркетинговые или партнерские сообщения:

```text
marketing_consent.md
privacy.md
bot_terms.md
```

Важно: согласие на рекламные сообщения должно быть отдельным от обязательных согласий, если рекламные сообщения не являются необходимой частью услуги.

---

## 6. Комплект документов для бота с CRM/API/webhook

Если бот связан с CRM, тикет-системой, API клиента, webhook или пользователь передает токены:

```text
data_processing.md
api_terms.md
privacy.md
terms.md
```

Если API используется только внутренне между backend и ботом, `api_terms.md` может быть не нужен как публичный документ, но `data_processing.md` может понадобиться, если пользователь подключает внешние сервисы или передает токены.

---

## 7. Комплект документов для бота с банковскими данными

Если бот показывает банковские операции, остатки, выписки, финансовые сводки или подключается к банку:

```text
bank_data_processing.md
privacy.md
personal_data_consent.md
terms.md
bot_terms.md
```

Если банковская функция платная:

```text
offer.md
payment_and_refund.md
subscription_rules.md
```

Также нужно указать, работает ли сервис только на чтение или может инициировать платежи.

| Функция бота | Какие документы нужны | Комментарий |
|---|---|---|

| Регистрация через бота | terms.md, privacy.md, personal_data_consent.md, bot_terms.md | Нужно фиксировать согласия |
| Запрос телефона | privacy.md, personal_data_consent.md | Нужно описать цель обработки телефона |
| Запрос email | privacy.md, personal_data_consent.md | Нужно для уведомлений, оплаты, чеков |
| Прием заявок | terms.md, privacy.md, personal_data_consent.md | Нужно описать данные заявки и сроки хранения |
| Поддержка через бота | support_rules.md, privacy.md, bot_terms.md | Нужно описать обращения, сообщения и вложения |
| Прием файлов/фото/голосовых | bot_terms.md, privacy.md | Нужно описать пользовательский контент и сроки хранения |
| Сервисные уведомления | bot_terms.md, privacy.md | Не являются рекламными, если нужны для работы сервиса |
| Рекламные уведомления | marketing_consent.md, privacy.md | Нужно отдельное согласие и способ отписки |
| Оплата через бота | offer.md, payment_and_refund.md, bot_terms.md | Ссылки на документы до оплаты |
| Подписка через бота | subscription_rules.md, offer.md, payment_and_refund.md | Описать период, отключение, окончание доступа |
| WebApp внутри бота | privacy.md, cookie_policy.md, terms.md | Если есть cookies, localStorage, аналитика |
| CRM-интеграция | data_processing.md, privacy.md | Если данные передаются во внешнюю CRM |
| API клиента | api_terms.md, data_processing.md | Если клиент использует API или webhook |
| Банковские уведомления | bank_data_processing.md, privacy.md | Если есть операции, остатки, выписки |

## 9. Документы и интерфейсные точки

| Интерфейсная точка | Что добавить | Документы |
|---|---|---|
| Первое сообщение бота | Ссылки на документы и краткий текст принятия условий | terms, privacy, personal_data_consent, bot_terms |
| Кнопка «Принять условия» | Явное подтверждение перед регистрацией или функциями | terms, privacy, personal_data_consent, bot_terms |
| Меню бота | Пункт «Правовые документы» | Все актуальные документы |
| Команда `/start` | Ссылки на документы и описание обработки данных | bot_terms, privacy |
| Команда `/help` | Ссылка на поддержку и правила поддержки | support_rules |
| Форма заявки в боте | Текст согласия на обработку данных | privacy, personal_data_consent |
| Отправка файла/фото | Предупреждение не отправлять секреты | bot_terms, privacy |
| Оплата в боте | Ссылки на оферту, оплату, подписку | offer, payment_and_refund, subscription_rules |
| Подключение CRM/API | Предупреждение о токенах | data_processing, privacy |
| Настройка webhook | Ответственность за endpoint | api_terms, data_processing |
| WebApp/PWA | Чекбоксы, ссылки, cookie-политика | terms, privacy, personal_data_consent, cookie_policy |
| Отписка от рекламы | Команда или кнопка отказа | marketing_consent |

## 10. Рекомендуемые тексты для бота

### Первое сообщение

```text
Перед использованием сервиса ознакомьтесь с Пользовательским соглашением, Политикой конфиденциальности и Правилами использования бота. Продолжая работу с ботом, вы подтверждаете согласие с условиями сервиса.
```

### Кнопка принятия

```text
Принять условия
```

### Регистрация с персональными данными

```text
Для регистрации в сервисе нужно обработать ваши данные. Продолжая регистрацию, вы даете согласие на обработку персональных данных и принимаете документы сервиса.
```

### Оплата

```text
Нажимая «Оплатить», вы принимаете условия Публичной оферты, Правил оплаты и возврата и Правил подписки, если они применимы.
```

### Рекламные сообщения

```text
Вы можете отдельно согласиться получать информационные и рекламные сообщения. Это согласие необязательно и может быть отозвано в любой момент.
```

### Отписка

```text
Вы отписались от рекламных сообщений. Сервисные уведомления, необходимые для работы сервиса, могут продолжать поступать.
```

## 14. Риски для юридической проверки

| Риск | Почему важно |
|---|---|
| Нет явного принятия документов в боте | Сложно доказать согласие пользователя |
| Не описаны Telegram/MAX ID и chat_id | Политика конфиденциальности не отражает фактические данные |
| Не описаны сообщения и вложения | Риск по пользовательскому контенту и срокам хранения |
| Нет отдельного рекламного согласия | Риск при маркетинговых рассылках |
| Рекламные и сервисные уведомления смешаны | Пользователь не понимает, от чего отписывается |
| Нет ссылки на оферту перед оплатой | Риск при платных функциях |
| Нет правил подписки при trial/подписке | Риск споров по доступу и окончанию периода |
| Нет описания зависимости от мессенджера | Риск претензий при сбоях Telegram/MAX |
| Не описана поддержка через бота | Непонятны сроки реакции и ограничения поддержки |
| Не фиксируются версии документов | Сложно доказать, какие условия принял пользователь |

## 15. Итоговый вывод

Для Demo Bot Service базово нужны:

```text
terms.md
privacy.md
personal_data_consent.md
bot_terms.md
support_rules.md
```

Если есть сайт, WebApp, PWA или аналитика, добавить:

```text
cookie_policy.md
```

Если есть оплата:

```text
offer.md
payment_and_refund.md
```

Если есть подписка, тарифы, trial или льготный период:

```text
subscription_rules.md
```

Если есть рекламные сообщения:

```text
marketing_consent.md
```

Если есть CRM/API/webhook или пользовательские токены:

```text
data_processing.md
api_terms.md
```

Если бот получает банковские данные:

```text
bank_data_processing.md
```

Перед публикацией документов нужно уточнить функции бота, состав данных, наличие WebApp, оплаты, подписки, рекламных сообщений, внешних интеграций, сроки хранения сообщений и порядок удаления данных.

Документы являются черновиками и требуют юридической проверки перед публикацией.
