

# Пример заполненной анкеты проекта: Telegram/MAX-бот

## Назначение примера

Этот пример показывает, как AI-агент может заполнить анкету проекта для сервиса, который работает через Telegram-бота, MAX-бота или другой мессенджер.

Пример подходит для проектов, где бот используется для:

- регистрации пользователей;
- приема заявок или заказов;
- отправки сервисных уведомлений;
- поддержки пользователей;
- подключения к личному кабинету или PWA;
- оплаты услуг или подписки;
- передачи статусов и сообщений;
- интеграции с внешними системами.

Пример не является готовым юридическим документом. Все данные ниже демонстрационные.

## 9. Интеграции и внешние сервисы

| Сервис | Тип интеграции | Какие данные передаются | Какие данные получаются | Комментарий |
|---|---|---|---|---|
| Telegram | Bot API / webhook | команды, сообщения, ID пользователя | сообщения, callback, статусы | Основной канал бота |
| MAX | Bot API / webhook | команды, сообщения, ID пользователя | сообщения, callback, статусы | Если используется |
| CRM | API | заявки, контакты, статусы | статусы сделок, ответы операторов | Уточнить CRM |
| Тикет-система | API | обращения, сообщения, вложения | статусы, ответы | Если есть поддержка |

| Email/SMS/push-провайдер | API | контакты, уведомления | статусы доставки | Если используются дополнительные каналы |
| Сервис аналитики | events/API | технические события | отчеты аналитики | Если есть сайт/WebApp |

| Правила оплаты и возврата | уточнить | Нужны, если есть оплата |
| Политика cookies | уточнить | Нужна, если есть сайт, WebApp, PWA, аналитика или cookies |
| Правила подписки | уточнить | Нужны, если есть подписка, trial или платные тарифы |
| Правила поддержки | да | Бот используется для поддержки или обращений |
| Согласие на рекламные сообщения | уточнить | Нужно, если есть рекламные рассылки через бот/email/push |
| Документ об обработке API-токенов | уточнить | Нужен, если пользователь подключает CRM, API, webhook или внешние сервисы |
| Документ об обработке банковских данных | нет / уточнить | Нужен, если бот получает банковские данные |
| Условия использования API | уточнить | Нужны, если сервис предоставляет API клиентам |
| Правила использования бота | да | Специальный документ для команд, уведомлений, данных мессенджера и ограничений |

## 17. Риски для юридической проверки

| Риск / вопрос | Комментарий |
|---|---|
| Пользователь не принимает документы до начала использования бота | Нужно добавить первое сообщение или кнопку принятия |
| Нет отдельного согласия на обработку персональных данных | Риск при регистрации и сборе телефона/email |
| Рекламные сообщения смешаны с сервисными | Нужно отдельное marketing_consent |
| Не описаны Telegram/MAX ID и chat_id | Нужно добавить в privacy |
| Не описано хранение сообщений и вложений | Нужно указать сроки хранения |
| Нет правил отключения бота | Пользователь не понимает, как прекратить уведомления |
| Нет ссылки на оферту перед оплатой | Риск при платных функциях |
| Не описана зависимость от Telegram/MAX | Нужно ограничение ответственности |
| Не описаны внешние интеграции | Нужно data_processing при CRM/API/webhook |
| Не фиксируются версии принятых документов | Сложно доказать согласие пользователя |

## 18. Итоговый вывод агента

Проект относится к сервисам, работающим через Telegram/MAX-бота, с возможной регистрацией, уведомлениями, поддержкой, WebApp/PWA, оплатой и внешними интеграциями.

Базовый комплект документов:

- Пользовательское соглашение;
- Политика конфиденциальности;
- Согласие на обработку персональных данных;
- Правила использования бота;
- Правила поддержки.

Дополнительно нужны при наличии соответствующих функций:

- Публичная оферта — если есть платные услуги;
- Правила оплаты и возврата — если есть оплата;
- Правила подписки — если есть подписка, тарифы или пробный период;
- Политика cookies — если есть сайт, WebApp, PWA или аналитика;
- Согласие на рекламные сообщения — если есть рекламные рассылки;
- Правила обработки API-токенов и данных интеграций — если есть CRM/API/webhook;
- Правила обработки банковских данных — если бот получает банковские данные;
- Условия использования API — если сервис предоставляет API клиентам.

Перед публикацией документов нужно уточнить владельца сервиса, состав данных, функции бота, наличие оплаты, подписки, рекламных сообщений, WebApp/PWA, интеграций и сроки хранения сообщений.

Подготовленные документы должны быть проверены владельцем проекта и юристом перед публикацией.legal-documents-matrix.example.md
