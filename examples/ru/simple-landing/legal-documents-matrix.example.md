

# Пример матрицы правовых документов: простой лендинг

## Назначение примера

Этот пример показывает, как AI-агент может определить комплект правовых документов для простого лендинга, сайта-визитки или промо-страницы.

Пример основан на демонстрационной анкете проекта:

```text
examples/ru/simple-landing/project-questionnaire.example.md
```

Все данные ниже демонстрационные. Итоговый комплект документов для реального проекта должен быть проверен владельцем проекта и юристом.

|---|---|

## 2. Итоговая матрица документов

| Документ | Нужен | Причина | Какие данные нужны |
|---|---|---|---|
| Пользовательское соглашение | нет / уточнить | Для простого лендинга без регистрации и личного кабинета обычно не требуется. Может понадобиться, если есть правила использования сайта, пользовательский контент или сложная услуга | Нужно уточнить, есть ли интерактивные функции, личный кабинет, публикация контента или специальные ограничения |
| Политика конфиденциальности | да | Лендинг собирает персональные данные через форму заявки, а также может использовать cookies, аналитику и логи | Владелец, реквизиты, состав данных, цели обработки, CRM, аналитика, сроки хранения, третьи лица |
| Согласие на обработку персональных данных | да | Форма заявки собирает телефон, email, имя или комментарий | Поля формы, цели обработки, срок действия согласия, порядок отзыва |
| Публичная оферта | нет | На лендинге нет онлайн-оплаты и продажи услуги через сайт | Если появится оплата или кнопка покупки, документ станет нужен |
| Правила оплаты и возврата | нет | Нет платежной формы и приема оплаты | Если появится онлайн-оплата, документ станет нужен |
| Правила подписки | нет | Нет подписки, тарифа, trial или автопродления | Если появится подписочная модель, документ станет нужен |
| Политика cookies | да / уточнить | Нужна, если используются cookies, аналитика, рекламные пиксели, localStorage или счетчики | Список счетчиков, cookies, пиксели, модель cookie-баннера, отключение cookies |
| Правила поддержки | нет / уточнить | Обычно не нужны для простого лендинга. Нужны, если сайт заявляет поддержку, сроки реакции или отдельные условия консультаций | Каналы поддержки, время работы, сроки реакции, что входит/не входит |
| Согласие на информационные и рекламные сообщения | уточнить | Нужно, если есть email/SMS/push/мессенджер-рассылка рекламного характера | Каналы рассылки, виды сообщений, способ отписки, фиксация согласия |
| Правила обработки API-токенов и данных интеграций | нет | Посетители лендинга не передают API-токены и не подключают внешние сервисы | Если появятся пользовательские интеграции или API-токены, документ станет нужен |
| Правила обработки банковских данных | нет | Лендинг не получает банковские данные | Если появится подключение банка или финансовые данные, документ станет нужен |
| Условия использования API | нет | Лендинг не предоставляет клиентский API | Если появится публичный API, документ станет нужен |
| Правила использования бота | уточнить | Нужны, если есть полноценный Telegram/MAX-бот, а не просто ссылка на чат или канал | Ссылка на бота, функции, команды, данные мессенджера, уведомления |

## 3. Минимальный комплект документов для простого лендинга

Если на лендинге есть только форма заявки и аналитика, минимальный комплект:

```text
privacy.md
personal_data_consent.md
cookie_policy.md
```

Если cookies и аналитика не используются, `cookie_policy.md` может быть не нужна, но это нужно подтвердить технически.

Если есть рекламная рассылка:

```text
marketing_consent.md
```

Если есть полноценный бот:

```text
bot_terms.md
```

Если на лендинг добавляется оплата:

```text
offer.md
payment_and_refund.md
```

## 4. Связь функций лендинга с документами

| Функция лендинга | Какие документы нужны | Комментарий |
|---|---|---|
| Форма заявки | privacy.md, personal_data_consent.md | Нужно согласие до отправки имени, телефона, email или комментария |
| Форма обратной связи | privacy.md, personal_data_consent.md | Аналогично форме заявки |
| Форма подписки на новости | privacy.md, marketing_consent.md | Если сообщения рекламные или маркетинговые, нужно отдельное согласие |
| Cookies сайта | cookie_policy.md, privacy.md | Если есть технические или аналитические cookies |
| Яндекс.Метрика / Google Analytics / Top.Mail.Ru | cookie_policy.md, privacy.md | Указать счетчики, цели, события, передачу третьим лицам |
| Рекламные пиксели | cookie_policy.md, privacy.md | Указать рекламные системы, ретаргетинг и события |
| UTM-метки | privacy.md, cookie_policy.md | Если сохраняются в заявках или аналитике |
| CRM для заявок | privacy.md | Указать передачу данных в CRM как третьему лицу или обработчику |
| Email-уведомления по заявкам | privacy.md | Указать email-провайдера, если используется |
| Кнопка перехода в Telegram/MAX | privacy.md / bot_terms.md | Если просто ссылка — достаточно privacy; если полноценный бот — нужен bot_terms |
| Онлайн-чат | privacy.md, cookie_policy.md | Виджет может собирать cookies, IP, сообщения |
| Капча | privacy.md, cookie_policy.md | Капча может передавать технические данные стороннему сервису |
| Онлайн-оплата | offer.md, payment_and_refund.md, privacy.md | Если появится платежная форма |
| Платная подписка | offer.md, payment_and_refund.md, subscription_rules.md | Если появится подписочная модель |

## 5. Документы и интерфейсные точки

| Интерфейсная точка | Что добавить | Документы |
|---|---|---|
| Футер сайта | Ссылки на документы | privacy, personal_data_consent, cookie_policy |
| Форма заявки | Чекбокс или текст согласия | privacy, personal_data_consent |
| Форма обратной связи | Чекбокс или текст согласия | privacy, personal_data_consent |
| Форма подписки на новости | Отдельный чекбокс согласия | marketing_consent, privacy |
| Cookie-баннер | Уведомление или настройки cookies | cookie_policy |
| Страница «Правовые документы» | Список всех документов | Все актуальные документы |
| Кнопка мессенджера | Пояснить переход во внешний сервис, если нужно | privacy, bot_terms при полноценном боте |
| Страница спасибо / успех заявки | Информация об обработке заявки и поддержке | privacy / support_rules при необходимости |

## 6. Рекомендуемые тексты для лендинга

### Форма заявки с отдельным чекбоксом

```text
[ ] Я даю согласие на обработку персональных данных для обработки моей заявки и обратной связи
```

Рядом или внутри текста должны быть ссылки на:

```text
Политику конфиденциальности
Согласие на обработку персональных данных
```

### Форма заявки без чекбокса, менее строгий вариант

```text
Нажимая «Отправить», я даю согласие на обработку персональных данных для обработки моего обращения и соглашаюсь с Политикой конфиденциальности.
```

Для более надежного сценария лучше использовать отдельный чекбокс.

### Подписка на новости

```text
[ ] Я согласен получать информационные и рекламные сообщения
```

Этот чекбокс должен быть необязательным, если рассылка не нужна для обработки заявки.

### Cookie-уведомление

```text
Мы используем cookies и похожие технологии для работы сайта, аналитики и улучшения сервиса. Подробнее — в Политике cookies.
```

Если используются рекламные cookies, может потребоваться более строгий баннер с выбором категорий.

## 7. Какие согласия нужно хранить

| Сценарий | Типы документов / согласий | Источник |
|---|---|---|
| Отправка формы заявки | personal_data_consent, privacy | landing_form |
| Отправка формы обратной связи | personal_data_consent, privacy | feedback_form |
| Подписка на новости | marketing_consent | subscription_form |
| Отзыв рекламного согласия | marketing_consent, event_type=revoked | unsubscribe_link / manual |
| Принятие cookies, если реализовано | cookie_policy | cookie_banner |
| Переход к оплате, если добавится оплата | offer, payment_and_refund | payment |

Для простого лендинга журнал согласий может быть проще, чем для SaaS, но желательно хранить хотя бы:

```text
form_type
checkbox_text
document_version
accepted_at
ip_address
user_agent
source_url
```

## 10. Риски для юридической проверки

| Риск | Почему важно |
|---|---|
| Нет Политики конфиденциальности | Пользователь не видит правила обработки данных |
| Нет согласия под формой | Сложно подтвердить согласие на обработку данных |
| Чекбокс согласия отмечен заранее | Пользователь не выражает активное согласие |
| Нет отдельного рекламного согласия | Риск при email/SMS/мессенджер-рассылках |
| Используется аналитика, но нет cookie policy | Не раскрыты cookies и счетчики |
| Есть рекламные пиксели, но нет уведомления | Риск по рекламным cookies и ретаргетингу |
| Заявки передаются в CRM, но это не описано | Не раскрыта передача третьему лицу |
| В события аналитики передается телефон/email | Риск избыточной передачи персональных данных |
| Нет сроков хранения заявок | Документы неполные |
| На сайте появилась оплата, но нет оферты | Риск при продаже услуг через сайт |

## 11. Итоговый вывод

Для Demo Landing минимально нужны:

```text
privacy.md
personal_data_consent.md
cookie_policy.md
```

`cookie_policy.md` нужен, если используются cookies, аналитика, рекламные пиксели, localStorage, sessionStorage или похожие технологии.

Если есть рекламная рассылка, добавить:

```text
marketing_consent.md
```

Если есть полноценный Telegram/MAX-бот, добавить:

```text
bot_terms.md
```

Если на лендинг добавится онлайн-оплата, добавить:

```text
offer.md
payment_and_refund.md
```

Если появится подписка или trial, добавить:

```text
subscription_rules.md
```

Перед публикацией документов нужно уточнить владельца сайта, поля формы, CRM/рассылки, аналитику, рекламные пиксели, сроки хранения заявок, cookie-баннер и наличие мессенджер-бота.

Документы являются черновиками и требуют юридической проверки перед публикацией.
