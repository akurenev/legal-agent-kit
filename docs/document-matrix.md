

# Матрица выбора правовых документов

## Назначение

Этот файл помогает AI-агенту определить, какие правовые документы нужны проекту в зависимости от его функций, бизнес-модели, состава данных, платежей и интеграций.

Матрица не заменяет юридическую проверку. Она используется как практический ориентир при подготовке черновиков документов.

---

## Как использовать матрицу

Агент должен:

1. Изучить описание проекта и доступные файлы.
2. Заполнить `LEGAL_PROJECT_QUESTIONNAIRE.md`.
3. Найти в матрице все условия, которые относятся к проекту.
4. Сформировать итоговый комплект документов.
5. Объяснить, почему каждый документ нужен.
6. Если функция проекта неочевидна — оставить пометку `[УТОЧНИТЬ]` и добавить вопрос в список уточнений.

Важно: если к проекту относится несколько условий, документы объединяются в один комплект без дублей.

## 1. Базовая матрица

| Условие проекта | Рекомендуемые документы | Что проверить дополнительно |
|---|---|---|
| Есть только информационный сайт без форм | Политика cookies, если используются cookies или аналитика | Есть ли аналитика, счетчики, пиксели, встроенные виджеты |
| Есть лендинг с формой заявки | Политика конфиденциальности, Согласие на обработку персональных данных, Политика cookies | Есть ли чекбокс согласия под формой |
| Есть форма обратной связи | Политика конфиденциальности, Согласие на обработку персональных данных | Какие данные собираются: имя, телефон, email, комментарий |
| Есть форма подписки на новости | Политика конфиденциальности, Согласие на обработку персональных данных, Согласие на информационные и рекламные сообщения | Можно ли отписаться от рассылки |
| Есть регистрация пользователя | Пользовательское соглашение, Политика конфиденциальности, Согласие на обработку персональных данных | Фиксируется ли факт принятия документов |
| Есть личный кабинет | Пользовательское соглашение, Политика конфиденциальности, Согласие на обработку персональных данных | Есть ли роли, права доступа, удаление аккаунта |
| Есть пользовательские настройки | Пользовательское соглашение, Политика конфиденциальности | Какие данные пользователь может изменить или удалить |
| Есть загрузка файлов пользователем | Пользовательское соглашение, Политика конфиденциальности, Правила пользовательского контента | Кто отвечает за содержимое файлов |
| Есть пользовательский контент | Пользовательское соглашение, Правила размещения и модерации контента | Есть ли жалобы, удаление, блокировка |
| Есть платные услуги | Публичная оферта, Правила оплаты и возврата | Понятен ли момент оказания услуги |
| Есть онлайн-оплата | Публичная оферта, Правила оплаты и возврата, Политика конфиденциальности | Платежный провайдер, фискализация, чеки |
| Есть подписка | Публичная оферта, Правила подписки, Правила оплаты и возврата | Период, продление, отключение, окончание доступа |
| Есть пробный период | Публичная оферта, Правила подписки | Условия перехода с пробного периода на платный |
| Есть автопродление | Публичная оферта, Правила подписки, Правила оплаты и возврата | Явно ли указаны условия повторных списаний |
| Есть льготный период | Правила подписки, Публичная оферта | Что доступно в льготный период и когда доступ прекращается |
| Есть тарифы | Публичная оферта, Правила подписки | Нет ли противоречий между тарифами на сайте и офертой |
| Есть индивидуальные условия | Публичная оферта или отдельный договор | Нужно ли указать приоритет индивидуального договора |

## 9. Правило сомнения

Если агент не уверен, нужен ли документ, он должен:

1. Не исключать документ молча.
2. Указать документ как `[УТОЧНИТЬ]`.
3. Объяснить, от какого условия зависит необходимость документа.
4. Добавить вопрос в `legal_questions.md`.

Пример:

```text
Правила подписки: [УТОЧНИТЬ]
Причина: в описании проекта указаны тарифы, но не ясно, предоставляется ли доступ по подписке или разовой оплатой.
Вопрос: доступ к сервису оплачивается разово или предоставляется на ограниченный период?
```

## 10. Финальная проверка матрицы

Перед подготовкой документов агент должен проверить:

- все ли функции проекта учтены;
- не пропущены ли формы сбора данных;
- есть ли оплата или подписка;
- есть ли cookies, аналитика или рекламные системы;
- есть ли API, webhook или интеграции;
- есть ли API-токены или банковские данные;
- есть ли Telegram/MAX-бот;
- есть ли поддержка пользователей;
- есть ли пользовательский контент;
- есть ли внешние сервисы, от которых зависит результат;
- сформирован ли итоговый комплект документов без дублей;
- подготовлены ли вопросы по неясным условиям.
